Mediatyper

Hva er medietyper?

Mediatyper ble orginalt innført i HTML4 og gjorde det mulig å definere stiler for forskjellige mediatyper. Media referer til hvilken type apparat som blir brukt for å vise websiden. For øyeblikket finnes det fire mediatyper som det fortsatt er i bruk; all, screen, print og speech[1]. Disse blir nok også utfaset etter hvert da det nye systemet med media queries er bedre egnet i utvikling av responsivt design.

Hvordan brukes de?

Man kan lage egne CSS-filer for de forskjellige medietypene, for så å linke til dem til HTML-documentet på vanlig måte. Man refererer da til hvilken mediatype man ønsker det skal gjelde for i linken.[2] Eksempel HTML-dokument:

<link rel="stylesheet" media="print" href="print.css" />

Når nettsiden lastes inn sjekkes det hvilken mediatype som brukes og det rette stilarket blir brukt.

De kan også skrives rett inn i CSS-filen med bruk av @media som ble introdusert med media queries. Eksempel CSS-fil:

@media print { body { font-size: 11pt; } } @media screen { body { font-size: 16px; } }

Hvilke fordeler og ulemper har dette?

Bedre oversikt over hvilke apparater som har støtte når man holder koden adskilt med å bruke flere stilark. Endringene som skal til for god visning på de enkelte mediatypene kan være omstendelig og et stilark kan fort bli uoversiktlig.

Med mediatyper kan man gjøre opplevelsen av siden god for alle uansett hvilken måte de velger å vise siden på.

Spesielt når man skal skrive ut noe fra en nettside er det nyttig å kunne skille mellom hva som er viktig å få med seg. Er for eksempel ikke viktig å ha med seg menyen til nettsiden på et utskrevet dokument.

Det finnes ikke mediatyper som dekker alle typen apparater der ute, så man måtte uansett ha spesifisert for kunne gi det beste resultatet. Dette gjøres bedre i media queries.

I de tilfellene det ikke skal så mye til for å dekke behovet for de forskjellige mediatypene. Er det bedre å holde all CSS-kode i samme fil. Gjennom media queries er dette nå mulig med @ regelen.

Hvorfor brukes ikke alle typer like mye som tidligere?

Mediatyper ble laget for å gjøre det enklere å stilsette websider for forskjellige apparater, men etter at media queries kom ut med CSS3 ble det enklere å bare gjøre det på den måten. Det finnes utallige apparater med forskjellige størrelser og egenskaper der ute og istedenfor å lage en mediatype til hver enkelt, brukes nå media queries isteden.

De mediatypene som ble fjernet[3] i siste oppdatering er:

"tty"
Brukes på mediatyper som har begrenset mulighet for visning av innhold. For eksempel terminaler.
"tv"
For bruk på tv-er. Da disse før hadde lav oppløsning, dårligere farger, osv.
"projection"
Prosjektorer og lignenende.
"handheld"
Mobiltelefoner, tabber og andre enheter med små skjermer og begrenset bandwidth.
"braille"
For å lage blindeskrift.
"embossed"
For å skrive ut sidene i blindeskrift. Utskriftsfunksjonen til braille.
"aural"
For mediatyper med talesyntese som er et tekst- til talesystem. Denne er nå byttet ut med "speech".

Disse skal ikke brukes og utviklere anbefales å velge hensiktsmessige media features (Media feature er en mer fin-justert test enn mediatyper. Den tester en spesifikk egenskap hos brukere eller apparatet) ut fra apparatet de prøver å stilsette.
