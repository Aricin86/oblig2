Git/GitHub

Hva er Git?

Git er verdens mest populære versjonskontrollsystem.[1]

Git er et versjonskontrollsystem (VCS – version control stystem) for å holde styr på endringene og de forskjellige versjonene av en eller flere filer. Et versjonskontrollsystem er programvare som i all hovedsak blir brukt til å administrere kildekode i utviklingen av programvare. Det finnes mange slike programmer og som med IDE (integrated development eviroment), velger man den man liker best. Noen av de mest kjente er TFS (Microsoft Team Fundation Server), Helix Core, Subversion, AWS CodeCommit, Mercurial, Fossil og Plastic SCM.

Git har åpen kildekode (open-source). Det betyr at den originale koden er tilgjengelig for alle og at man kan endre og dele den på nytt. Git ble startet av Linus Trovalds, en av de mest kjente programvareutviklerne i verden. Han er blant annet grunnleggeren av Linux, som også har åpen kildekode. Git er bygget for å virke med Linux kernel, som betyr at den måtte ta hånd om store repositories. Det er skrevet i et språk kalt C, som reduserer tiden det tar å utføre kommandoer. Git er bygget med tanke på fart og ytelse.

Det finnes forskjellige typer versjonskontroller, men de to største bolkene kommer under distribuert versjonskontroll og klient-tjener systemet. Git er en distribuert versjonskontroll. Det betyr at den komplette kodebasen, inkludert hele endringshistorikken, ligger på alle datamaskinene som er knyttet til det prosjektet.

Hva er GitHub?

"GitHub is a code hosting platform for version control and collaboration. It lets you and others work together on projects from anywhere."[2]

GitHub er bygget på versjonskontrollsystemet til Git. Det vil si at versjonskontroll er implementert på siden og at man får god kontroll på prosjekter og filer. Det gjør det også enkelt å samarbeide med andre på prosjekter uansett hvor de er i verden. En av de største attributtene til GitHub er muligheten for nettverksbygging. GitHub er en møteplass der folk deler, lærer og jobber sammen for å lage programvare.

Noen arbeidsgivere spør søkere om de har noe liggende på GitHub og dette kan telle positivt i forbindelse med jobbsøking. Det er gratis å bruke GitHub for offentlige prosjekter og prosjekter med åpen kildekode.

For å lage ett nytt prosjekt på GitHub oppretter man et repository[3]. Repository er kjernen av et versjonskontrollsystem, som oppbevarer prosjektets datastruktur og metadata. Den kan inneholde filer, bilder, videoer eller hva som helst annet man trenger i prosjektet.

Det er mulig å lage forgreininger for å lage endringer som ikke påvirker hovedprosjektet ("forking"/"brancing"). Etter at man har gjort endringer og ser at disse er nyttige og stabile fletter man dem inn i hovedprosjektet igjen med å "hente den inn". Dette gjøres ved å lage en "pull request" som åpner en dialog med forfatterne av prosjektet og det er enkelt å ha en diskusjon om hvorvidt endringene som er gjort er klare for å tas inn i hovedprosjektet.

Hvilke fordeler er det med å bruke Git?

Siden Git er en distribuert versjonskontroll og alle datamaskinene har hele repository nedlastet, er det en smal sak hvis en server dør eller man mister datamaskinen. Det vil da bare være å laste inn prosjektet fra en av de andre datamaskinene. Alle har en klonet versjon av prosjektet og en full backup av dataen, uavhengig av nettverkstilgang og sentrale servere.

I Git har man mulighet til å gjøre noe kalt branching (forgreining). Når man lager en forgreining går man ut av master (hovedlinjen), men med den som grunnlag, og kan teste nye ting uten at det har effekt på hovedlinjen for utviklingen. Man kan bytte mellom å jobbe med de forskjellige forgreiningene og når man er fornøyd gjør man en merge (fusjon). Man fletter da sammen koden fra forgreiningen med hovedlinjen. Dette gjør at utviklingen av et program ikke trenger å være lineær, men støtter at man kan jobbe mer fleksibelt med flere deler samtidig.

Min vurdering av Git

Førsteinnskytelsen er at Git ser ut som et utrolig nyttig verktøy for utvikling av programvare og andre type prosjekter. Etter å ha lest, opprettet min egen bruker og lastet ned Git føles det litt i overkant å bruke Git til et så lite prosjekt som dette, men for læringen sin skyld er det perfekt. Har ikke satt meg inn i kommandoskriving i Git og kjører heller desktop-versjonen, da tiden rett og slett ikke strekker til denne gangen.
